Initialization, Condition, Update

  • Loops are defined by two components
    • Condition (when is the body executed)
    • Body (what is executed, causes condition to change from true to false if it is ever to end)
  • Consequently all loops go through
    • Initialization setup the variables used in the condition expression (before the first condition check)
    • Condition checking evaluate the condition expression
    • Update: change the values in some or all of the variables used in the condition (after every condition check)

Common Loops

  • Counter Controlled
    • Repeat x number of times
  • Sentinel Controlled
    • Repeat until value x is entered
  • Endfile Controlled
    • Repeat until all the file is scanned
  • Input Validation Loop
    • Repeat until a value is within acceptable range

C standards and for

  • A C standard defines the C language exactly
    • Syntax, Grammar, and Vocabulary
  • GCC uses the GNU89 standard by default
    • Yes, that's 1989
    • Declarations in the initialization were not around in 1989, so the following code does not compile
    • for( int i =0; i<10; i=i+1)
    • If your code uses this you can either declare i before the initialization or you can change the standard gcc to C99 by
    • gcc -std=c99 -0 executable source.c

New Operators I

  • Arithmetic assignment operators
    • Combind an arithmetic operators with the assignment operator
    • Preform an arithmetic operation on the variable in the left side of the xpression and assign the value of the expression to that variable.
    • 1+=t is equivalent to i = i+5
    • += -= *= /= %=

New Operators II

  • Prefix ++ Prefix --
    • ++i
  • Postfix ++, Postfix --
    • i++
  • All of these operators will change the value of the variable i by 1
  • They have high precedence
    • higher than any other operators we've seen
    • Postfix has higher precedence than prefix

New Opperators III

  • ++i
    • Side effect: increases the value of i by 1
    • evaluates to original value of i plus 1
  • --i
    • Side effect: decreases the value of i by 1
    • Evaluates to original value of i minus 1
  • i++
    • Side effect: increases the value of i by 1
    • Evaluates to original value of i
  • i--
    • Side effect: decreases the value of i by 1
    • evaluates to original value i

Warning

  • Using these causes the variables to be changed
  • When used within a larger expression, a single variable may carry different values depending on when the compile decides operators are evaluated

    • int x = f;
